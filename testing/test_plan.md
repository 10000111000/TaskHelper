# План Тестирования

## 1\. Цели Тестирования 

Цель тестирования - обеспечить высокое качество и надежность основных функциональных возможностей бота, а также проверить удобство взаимодействия с пользователем.

| **Критерий** | **Проверяемая Область** | **Описание Цели** |
| :-- | :-- | :-- |
| **Функциональность** | CRUD-операции, Напоминания | Убедиться, что все команды (/add, /list, редактирование, удаление) работают по спецификации, а напоминания срабатывают точно и своевременно. |
| **Интеграция** | AI (GigaChat) | Проверить, что бот корректно взаимодействует с внешним AI-сервисом, надежно парсит структурированный JSON и обрабатывает конфликты форматов данных. |
| **Обработка Ошибок** | Валидация, Критические сбои | Проверить, что бот корректно реагирует на некорректный ввод (например, неверный формат даты) и не падает при критических ошибках (например, ошибка подключения к БД или AI). |
| **Юзабилити (UX/UI)** | Интерфейс, Навигация | Оценить интуитивность главного меню, ясность FSM-диалогов и удобство использования клавиатур (Reply/Inline). |
| **Регрессия** | Устойчивость после изменений | Убедиться, что исправленные ошибки не внесли новые дефекты в ранее работавший функционал. |

## 2\. Инструменты и Среда Тестирования

| **Категория** | **Инструмент / Среда** | **Описание** |
| :-- | :-- | :-- |
| **Среда Исполнения** | Telegram Desktop / Mobile (iOS/Android) | Платформа, на которой пользователи взаимодействуют с ботом. |
| **Среда Разработки** | Python 3.9+, venv | Среда, в которой развернут код бота. |
| **База Данных** | MySQL / MariaDB | Тестирование подключения, сохранения и восстановления данных. |
| **AI-сервис** | GigaChat API | Проверка передачи данных и парсинга ответа. |
| **Инструменты Тестирования** | Ручное тестирование, Google Forms | Основной метод - ручное прохождение всех сценариев; Google Forms - для сбора UX-отзывов. |
